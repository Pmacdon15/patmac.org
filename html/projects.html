patchat.ca

PatChat is a web app that i made for fun. The websitePatChat.ca is a chatroom, that I host on my unbuntu server on my raspberry pi 4. I used express and socket.io to make the chatroom. I used pm2 to run the server in the background. I used apache and a vitural host point to the desired port. Apache all handles my ssl certificate. I used certbot to get the ssl certificate. I used a reverse proxy to point to my ip address. I used port forwarding to point all traffic on my ip address to my raspberry pi. Apache does the rest.

Requierments:

  • Raspberry Pi 4 running Unbunt Server
  • A computer
  • Your perfered IDE
  • Knowledge of Node.js, Express, Socket.io, Apache, and SSL certificates.

Process I followed:

  • Set up a Unbuntu server on a Raspberry Pi 4.
  • Set up port forwarding on router which will reserve an IP address for the Pi.
  • Install Node.js and Express on the Pi.
  • Write Express server with Socket.io
  • Write client side js code to connect to server.
  • Write client side js code to send and recieve messages.
  • Write client side js code to display messages.
  • Write client side js code to display users.
  • Write a simple html page to display the chatroom.
  • Create or push a commit to a Github repository.
  • Clone the repository from GitHub to the Pi.
  • Download and set up PM2 so server can run in the background
  • Download and set up Apache
  • Set up a virtual host profile so that you can host more than one domain name with one ip.
  • Usert bot to set up an SSL certificate for your site.

Creating the chat room Seemed to go well afterI found a video breaking it down to its simplest components. Another layer of complexity was added when I allowed Apache to handle my ssl certificates as opposed to setting up the express server as an HTTPS server to begin with.I was able to get the site to work properly allowing Apache to handle the SSL certificatesalthough this method does generateone console error on the client sidedue toa connection failing at first.This does not affect the functionality of the chat room and I am still proud of the product I've produced.

Unbutu Server as an os for a Raspberry-Pi 4

This Allows me to have many Directories with different types of frameworks such as Apache, express, pytho or Rubyy.

Process I followed:

  • Download the Unbutu Server image from the official website. Look for the ARM 64 bit image.
  • Download the Raspberry Pi Imager from the official website. IF you dont already have it.
  • Use the Raspberry Pi Imager to flash the Unbutu Server image to the SD card. There is a gears menu that allows you to set the wifi and ssh, username and password up.
  • Insert the SD card into the Raspberry Pi.
  • Connect the Raspberry Pi to a monitor, keyboard, ethernet cable(if you havnt sent up the wifi in the imager).
  • Power on the Raspberry Pi.
  • Let the imager start up, when finished log in with the user name and password you set up in the imager.
  • Run the command sudo apt update && sudo apt upgrade.
  • Run the command sudo git -version to see if git is installed.
  • Downloand and set up your fravoite frame works and clone your favorite repositories.
    • I also install pm2 to make running servers in the back ground easy.

Requierments:

  • Raspberry Pi 4
  • Micro SD card
  • A computer to set up the Pi.

I have been using Unbutu server as my main os as an experiment. I have found it to be very stable and easy to use. and I have been able to set up many different types of servers on it. I have been able to set up apache, express, python and ruby servers. I use pm2 to run the servers in the background. I have also been able to set up an Express Downloads and uploads server. I have been able to download and upload files to the server from anywhere. To do this port forwarding must be set up.
